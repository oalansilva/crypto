# Escopo do MVP Cripto

## Hipotese

Se oferecermos um Monitor de estrategias de swing trade, alimentado por backtest e dados de mercado, para um grupo pequeno de investidores beta, entao pelo menos parte desses usuarios vai entender valor pratico suficiente para usar recorrente e sinalizar disposicao de pagar.

## Valor Central

Dar ao investidor uma leitura simples de oportunidade e risco para acompanhar estrategias, sem prometer lucro e sem vender sinal milagroso.

## Must-have para Beta Fechado

1. Login e acesso controlado.
2. Monitor como tela principal.
3. Estrategias favoritas vindas do fluxo de backtest ou selecao validada.
4. Cards com estado claro: `HOLD`, `WAIT`, `EXIT`, ou equivalente.
5. Distancia ate entrada, alvo, stop ou saida quando houver dado confiavel.
6. Grafico detalhado do ativo/estrategia.
7. Timeframes principais para swing trade: `4h` e `1d`.
8. Candles e oportunidades estaveis o suficiente para teste beta.
9. Disclaimer visivel: educacional, apoio a decisao, nao recomendacao financeira.
10. Teste ponta a ponta: login -> Monitor -> card -> grafico -> contexto/historico -> risco.

## Pode Ser Manual no Inicio

1. Selecao de beta testers.
2. Convite e onboarding.
3. Coleta de feedback.
4. Registro de metricas.
5. Suporte no Telegram.
6. Pergunta de disposicao de pagar.

## Fora do MVP Agora

1. Pagamento automatizado.
2. Comunidade automatizada.
3. Email marketing.
4. App publico aberto.
5. Infraestrutura maior sem gargalo comprovado.
6. Promessa de acerto dos sinais.
7. Features administrativas avancadas.

## Criterio de Go para Beta

- Login funciona para usuario autorizado.
- Usuario chega ao Monitor sem confusao.
- Monitor carrega estrategias/oportunidades sem travar.
- Grafico abre a partir do Monitor.
- Disclaimer esta visivel.
- Fluxo ponta a ponta foi validado.
- Ha pelo menos 3 beta testers definidos.
- Ha canal claro de feedback.

## Criterio de No-Go

Nao liberar beta se:

- oportunidades ou candles travarem o fluxo principal;
- o usuario puder interpretar a tela como promessa de lucro;
- acesso estiver publico sem controle;
- nao houver forma simples de coletar feedback;
- nao houver evidencia minima do fluxo ponta a ponta.

## Sucesso nos Primeiros 30 Dias

- 3 a 5 usuarios beta convidados.
- Pelo menos 3 usando o fluxo principal.
- Pelo menos 3 conversas reais de feedback.
- Pelo menos 1 sinal claro de disposicao de pagar ou indicacao para outro usuario.
- Lista objetiva do que precisa mudar antes de cobrar.
