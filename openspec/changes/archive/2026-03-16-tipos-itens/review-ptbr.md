# Proposal Review — tipos-itens

## Resumo

**Dois problemas abordados:**

1. **Visual:** Bugs precisam de destaque visual (cor diferente) no board
2. **Workflow:** Não existe fluxo padrão para criar bugs — QA não tem como criar

**Soluções:**

1. Adicionar estilo/coloração específica para cards tipo "bug"
2. Implementar fluxo de criação de bugs:
   - QA cria bug vinculado à story
   - Story só termina quando bugs resolvidos
   - Bugs passam por DEV → QA → homologação

**Próximo passo:** Aprovação → DEV → QA → Homologação.
